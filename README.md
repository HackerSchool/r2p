# r2p

C++/Qt

2 coisas importantes:
 * client-side apps
 * performance

### apps
 * http server on both sides
 * Windows
    * ligar RDP pelo registo
    * começar jogo
    * `PUSH` po raspi para ele se ligar ao stream
 * raspi
    * `PUSH` po windows para ele ligar RDP e começar o jogo
    * esperar e ligar o stream

### performance
 * raspi stack
    * versões e overclock
 * freerdp options
    * rfx, gfx
